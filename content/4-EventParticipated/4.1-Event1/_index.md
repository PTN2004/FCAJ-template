# Summary Report: “Cloud Architect”

### Event Information
- Event: Cloud Architect
- Format: Team-based competition
- Participants: 8 teams, each consisting of 5 members
- Topic: Cloud Computing and Cloud Architecture Design

### Event Objectives

- Deepen understanding of cloud computing fundamentals
- Enhance knowledge of cloud architecture design patterns
- Strengthen teamwork and communication skills
- Improve problem-solving abilities under time pressure
- Familiarize participants with competitive cloud architecture scenarios

### Event Overview

The competition consisted of multiple rounds where two teams competed by answering a set of questions with increasing difficulty. The team with the higher score advanced to the next round. In the event of a tie, an additional tiebreaker question determined the winner based on response speed.

Each team could strategically use two special skills during the competition:
- Minimal Risk: If the answer was incorrect, no points were deducted; if correct, only half of the question's points were awarded.
- Star of Hope: If the answer was correct, the score was doubled; if incorrect, the penalty was also doubled.

The questions covered various cloud-related topics, including cloud services, networking, storage, security, scalability, high availability, and architecture design principles.

### Key Learnings
Through this event, I gained a better understanding of:

- Core concepts of cloud computing and the service model.
- Best practices for designing scalable and reliable cloud architectures.
- Cloud security issues and trade-offs in architecture.
- The importance of selecting the right cloud services based on system requirements.
- The difficulty of AWS certification exams.

### Personal Reflection

The event was both engaging and educational. The competitive format encouraged participants to think critically and discuss different architectural approaches before answering. The special game mechanics added an interesting strategic element, requiring teams to balance confidence and risk.

Overall, the event provided a valuable opportunity to reinforce cloud knowledge while improving teamwork and communication skills.

### Conclusion

Overall, the Cloud Architect event was well organized and successfully combined learning with friendly competition. It helped participants review cloud architecture concepts, practice analytical thinking, and enhance collaboration in a team environment.
